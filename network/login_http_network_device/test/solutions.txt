문제 1. 세션 기반 인증 방식에 관한 문제  
세션 기반 인증 방식에서 클라이언트와 서버 간의 연결을 유지하기 위해 사용되는 것은 무엇인가요?  

a) 세션 ID  
b) 토큰  
c) IP 주소  
d) 쿠키  

**정답**: a) 세션 ID  
**해설**: 세션 기반 인증에서는 세션 ID를 사용하여 클라이언트와 서버 간에 연결 상태를 유지합니다. 이 ID는 서버에서 생성되어 클라이언트에게 전달되고, 이후의 요청에서 클라이언트는 이 세션 ID를 통해 인증을 유지합니다.

문제 2. 세션 ID는 어디에 저장되며, 무엇을 식별하나요?  

a) 서버에 저장되어 클라이언트의 유니크한 ID를 식별  
b) 클라이언트에 저장되어 사용자의 요청을 추적  
c) 웹 브라우저에 저장되어 사용자 정보를 암호화  
d) 데이터베이스에 저장되어 요청 시간을 추적  

**정답**: b) 클라이언트에 저장되어 사용자의 요청을 추적  
**해설**: 세션 ID는 클라이언트에 저장되어 서버와 클라이언트 간의 상호작용을 추적하는 데 사용됩니다. 이 정보는 보통 쿠키에 저장됩니다.

문제 3. 세션 기반 인증에서 '상태 없음(stateless)'의 의미는 무엇인가요?  

a) 서버는 요청마다 이전 상태를 기억하지 않음  
b) 클라이언트는 상태를 기억하지만 서버는 그렇지 않음  
c) 모든 요청에 대해 상태를 유지해야 함  
d) 세션 ID만으로 상태를 추적할 수 있음  

**정답**: a) 서버는 요청마다 이전 상태를 기억하지 않음  
**해설**: '상태 없음(stateless)'은 서버가 클라이언트의 이전 요청에 대한 상태를 기억하지 않음을 의미합니다. 세션 ID를 사용하여 상태를 추적하지만, 서버는 각 요청을 독립적으로 처리합니다.

문제 4. JWT 토큰의 구성 요소에 포함되지 않는 것은 무엇인가요?  

a) 헤더  
b) 페이로드  
c) 서명  
d) 클라이언트 IP 주소  

**정답**: d) 클라이언트 IP 주소  
**해설**: JWT 토큰은 헤더, 페이로드, 서명 세 부분으로 구성됩니다. 클라이언트 IP 주소는 JWT의 일부가 아니며, 인증과 관련된 추가 정보는 페이로드에 포함될 수 있습니다.

-----------------------------------------

문제 5. JWT 토큰에서 '페이로드'는 무엇을 포함하나요?  

a) 토큰 발급자, 유효기간 등 데이터  
b) 암호화된 비밀 키  
c) 클라이언트의 IP 주소  
d) 요청의 URI  

**정답**: a) 토큰 발급자, 유효기간 등 데이터  
**해설**: JWT의 페이로드에는 토큰의 발급자, 유효기간, 사용자의 정보와 같은 데이터를 포함할 수 있습니다. 이는 암호화되지 않으며, 서명으로 안전성을 보장합니다.

문제 6. JWT 토큰의 장점으로 옳지 않은 것은 무엇인가요?  

a) 서버에 별도의 인증 저장소가 필요 없다  
b) 데이터가 토큰 자체에 포함되어 경량화된다  
c) 직렬화와 역직렬화 비용이 매우 높다  
d) 서버 과부하를 줄일 수 있다  

**정답**: c) 직렬화와 역직렬화 비용이 매우 높다  
**해설**: JWT는 데이터를 토큰 자체에 포함하므로 별도의 서버 저장소가 필요 없습니다. 직렬화와 역직렬화 비용은 상대적으로 낮으며, 서버의 부하를 줄이는 데 도움이 됩니다.

문제 7. Access 토큰과 Refresh 토큰의 차이점으로 옳은 것은 무엇인가요?  

a) Access 토큰은 수명이 짧고 Refresh 토큰은 길다  
b) Access 토큰은 서버에서 저장되고 Refresh 토큰은 클라이언트에 저장된다  
c) Access 토큰은 데이터를 암호화하는 데 사용된다  
d) Access 토큰은 클라이언트에서 인증을 처리하고, Refresh 토큰은 서버에서 인증을 처리한다  

**정답**: a) Access 토큰은 수명이 짧고 Refresh 토큰은 길다  
**해설**: Access 토큰은 상대적으로 짧은 수명을 가지며, 사용자가 로그인을 유지할 수 있도록 하기 위해 Refresh 토큰을 사용하여 새로운 Access 토큰을 발급받습니다.

문제 8. HTTP 상태 코드 '200 OK'는 어떤 의미인가요?  

a) 클라이언트 요청이 잘못되었음을 알린다  
b) 요청이 성공적으로 처리되어 응답이 반환되었다  
c) 서버에서 내부 오류가 발생했다  
d) 리소스가 영구적으로 이동했다  

**정답**: b) 요청이 성공적으로 처리되어 응답이 반환되었다  
**해설**: '200 OK'는 요청이 성공적으로 처리되었음을 나타내는 상태 코드입니다. 이 코드가 반환되면 요청에 대한 응답이 정상적으로 전달됩니다.

문제 9. HTTP 상태 코드 '401 Unauthorized'는 어떤 상황에서 발생할까요?  

a) 서버의 리소스가 존재하지 않음  
b) 클라이언트 인증이 필요한데 인증되지 않았음  
c) 클라이언트 요청이 잘못되어 처리할 수 없음  
d) 서버가 요청을 이해할 수 없음  

**정답**: b) 클라이언트 인증이 필요한데 인증되지 않았음  
**해설**: '401 Unauthorized' 상태 코드는 클라이언트가 인증을 제공하지 않거나 잘못된 인증 정보를 제공한 경우에 발생합니다.

문제 10. HTTP 상태 코드 '502 Bad Gateway'는 무엇을 의미하나요?  

a) 요청이 잘못되었음  
b) 서버가 다른 서버에 요청을 보냈지만 응답을 받지 못함  
c) 클라이언트 인증이 필요함  
d) 요청에 대한 응답이 정상적으로 반환됨  

**정답**: b) 서버가 다른 서버에 요청을 보냈지만 응답을 받지 못함  
**해설**: '502 Bad Gateway'는 서버가 다른 서버에 요청을 보냈지만, 해당 서버에서 유효한 응답을 받지 못한 경우 발생합니다.

문제 12. L7 스위치의 역할은 무엇인가요?  

a) IP 주소와 포트를 기준으로 트래픽을 분배한다  
b) 로드 밸런싱을 통해 서버 부하를 분산시킨다

d) 데이터를 물리적 장비 간에 전송한다  

**정답**: b) 로드 밸런싱을 통해 서버 부하를 분산시킨다  
**해설**: L7 스위치는 애플리케이션 레벨에서 동작하며, URL, 헤더 등의 정보에 기반하여 로드 밸런싱을 통해 서버 부하를 분산합니다.

문제 13. L4 스위치와 L7 스위치의 차이점은 무엇인가요?  

a) L4는 IP 주소와 포트를 기반으로 트래픽을 분배하고, L7은 URL, 헤더 등 더 상세한 정보를 기반으로 분배한다  
b) L7은 패킷을 암호화하고, L4는 암호화하지 않는다  
c) L4 스위치가 로드 밸런싱을 하고, L7은 보안을 강화한다  
d) L4는 서버의 상태를 체크하고, L7은 URL을 관리한다  

**정답**: a) L4는 IP 주소와 포트를 기반으로 트래픽을 분배하고, L7은 URL, 헤더 등 더 상세한 정보를 기반으로 분배한다  
**해설**: L4 스위치는 네트워크 계층에서 IP 주소와 포트를 기준으로 트래픽을 분배하는 반면, L7 스위치는 애플리케이션 계층에서 URL, 헤더 등을 기반으로 트래픽을 분배합니다.

문제 14. 라우터의 주요 기능은 무엇인가요?

문제 15. L3 스위치는 L2 스위치와 무엇이 다른가요?  

a) L3 스위치는 IP 주소를 기준으로 라우팅을 하고, L2 스위치는 MAC 주소를 기준으로 트래픽을 전달  
b) L3 스위치는 데이터를 암호화하고, L2 스위치는 데이터를 해독한다  
c) L3 스위치는 네트워크를 외부와 연결하고, L2는 내부만 관리  
d) L3는 라우팅을 하지 않으며, L2만 라우팅을 담당한다  

**정답**: a) L3 스위치는 IP 주소를 기준으로 라우팅을 하고, L2 스위치는 MAC 주소를 기준으로 트래픽을 전달  
**해설**: L3 스위치는 IP 주소를 사용하여 네트워크 경로를 결정하고 라우팅하는 반면, L2 스위치는 MAC 주소를 사용하여 트래픽을 전달합니다.

보너스 2. 세션 기반 인증 방식에서 세션 ID를 클라이언트에게 전달하는 방법은 무엇인가요?  

a) HTTP 헤더  
b) 쿠키  
c) URL 파라미터  
d) 바디 내 데이터  

**정답**: b) 쿠키  
**해설**: 세션 ID는 일반적으로 쿠키에 저장되어 클라이언트와 서버 간의 통신에 사용됩니다. 쿠키는 세션 ID를 자동으로 전달하여 인증 상태를 유지합니다.
